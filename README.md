# 재활 운동 영상 앱 초안

안드로이드와 iOS에서 동작하는 앱을 목표로 한 초안입니다. 의사가 환자의 진단명을 등록해두면, 환자는 로그인 후 해당 진단명에 맞는 운동 영상만 볼 수 있는 구조입니다.

## 1) 역할 정의

### 관리자(운영자/의사)
- 환자 계정의 **사용 여부(활성/비활성)** 및 **사용 기간(시작일/종료일)** 설정
- 환자별 **진단명/진단코드** 추가, 삭제, 수정
- 환자 계정 생성 및 관리

### 환자
- 로그인 후 **지정된 운동 영상만 시청** 가능
- 검색, 업로드, 수정 등 다른 기능 없음

## 2) 핵심 흐름

1. 관리자가 환자를 등록하고 사용 기간을 설정한다.
2. 관리자가 환자의 진단명과 진단코드를 입력한다.
3. 환자는 앱에 로그인한다.
4. 앱은 환자 계정의 진단명/진단코드에 매칭되는 운동 영상을 표시한다.
5. 환자는 영상만 시청한다.

## 3) 데이터 구조(초안)

### 환자
- 환자ID
- 이름/연락처(선택)
- 로그인ID/비밀번호(또는 간편 로그인)
- 사용 여부(활성/비활성)
- 사용 기간(시작일, 종료일)
- 진단명/진단코드 목록

### 영상 콘텐츠
- 영상ID
- 제목
- 설명
- 진단명/진단코드 매핑
- 썸네일/영상 URL

## 4) 관리자 기능(초안)
- 환자 목록 조회
- 환자 상세 조회
- 환자 활성/비활성 전환
- 환자 사용 기간 설정
- 환자 진단명/진단코드 CRUD
- 진단명/진단코드-영상 매핑 관리

## 5) 환자 기능(초안)
- 로그인
- 진단명/진단코드에 맞는 영상 목록 보기
- 영상 재생

## 6) 진단명/진단코드 추가 방식
- 초기에는 관리자가 직접 입력
- 추후 **엑셀 파일 업로드**로 일괄 등록 가능하도록 확장

## 7) 다음 단계 제안
- 진단명/진단코드 표준화 방식 확정
- 영상 콘텐츠 분류 체계 확정
- 로그인 방식(일반/간편 로그인) 결정
- 관리 페이지(웹) 제공 여부 결정
